## ✅ Phase 4 — Protocol Maturity (Complete as of 2026-03-17)

### Done
- [x] RFC error shapes on all OIDC/OAuth endpoints (`{"error", "error_description"}`)
- [x] PKCE enforcement — public clients require `code_challenge=S256`
- [x] Refresh token validation — wrong-client binding, empty token → `invalid_grant`
- [x] RFC 7009 compliance — `/v1/oidc/revoke` returns 200 for unknown/expired tokens
- [x] Client authentication enforcement — confidential vs. public client rules
- [x] Scope hardening — `scopes_supported` in discovery, userinfo requires bearer
- [x] Stable claim guarantees — `sub` permanent, `email`/`name` scoped correctly
- [x] `paused` → `suspended` rename everywhere (DB migration 0040, Rust, TypeScript, UI)
- [x] Platform workspace lock (`platform_locked` boolean, migration 0039)
- [x] Clear suspended error message at login (`"This account has been suspended."`)
- [x] Admin console column alignment + shared `StatusBadge` component
- [x] Full test suite: 40 files, ~297 requests

### Known Gap (tracked, still open)
- [ ] `end_session_endpoint` — RP-initiated logout — **first item in Phase 5**

## ✅ Phase 5 — Security Architecture (Complete — 2026-03-17)

*Goal: push Security from 8.5 → 10/10.*

### Logout
- [x] `end_session_endpoint` (`GET /v1/oidc/end-session`) — RP-initiated logout
  - Clears session cookie, revokes session in DB
  - Redirects to `post_logout_redirect_uri` with optional `state` param
  - Advertised in `/.well-known/openid-configuration`
- [ ] Front-channel logout (iframe broadcast, optional per client) — **Phase 7+**
- [ ] Back-channel logout (server POST, optional per client) — **Phase 7+**

### Identity Lifecycle Safety
- [x] **Secure email change flow** — two-step: request → verify; blocks if email taken; notifies old address
- [x] **Provider unlink safety** — cannot unlink last login method; unsupported providers → 400
- [ ] Account recovery (admin reset flow) — **Phase 7+**

### Key Lifecycle
- [x] **OIDC signing key rotation** — `POST /v1/admin/signing-keys/rotate`; DB-backed JWKS with 24h rollover window; `kid` in all JWTs

### Session & Device Security (deferred)
- [ ] Device trust — **Phase 7+**
- [ ] Step-up auth (`acr_values`) — **Phase 7+**
- [ ] Anomaly scoring — **Phase 7+**

## ✅ Phase 6 — Control Plane Maturity (Complete — 2026-03-17)

*Goal: push Control plane from 8 → 10/10.*

### Policy Clarity
- [x] Effective policy view (`GET /v1/orgs/current/effective-policy`)
- [x] Policy change preview (`POST /v1/orgs/current/auth-policy/preview`) — lockout count warning
- [x] Self-lockout prevention (`POST /v1/orgs/current/auth-policy/self-check`)
