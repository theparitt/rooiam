# OpenAPI + SDK — Phased Plan

Status: **In progress — Phases A, B, C done (2026-06-02). Next: Phase D (browser SDK).**

Goal: a typed, language-agnostic SDK layer so downstream apps integrate with
Rooiam through generated clients, not hand-rolled `fetch`. The server's OpenAPI
spec (generated by `utoipa`, so it cannot drift) is the single source of truth.

Background design: [41_sdk_plan.md](./41_sdk_plan.md) (two TS packages, monorepo,
generate-from-code). This doc is the **execution plan** with phases.

Hard rule (locked): **the SDK must be 100% stable + tested against the live
server before any working frontend (rooiam-admin, rooiam-app) is refactored onto
it.** Prove the layer first; move the apps last.

### The annotation recipe (mechanical — repeat per endpoint)

The 30 remaining endpoints are pure repetition of three proven shapes. For each:

1. Make the handler **`pub`** (utoipa's generated `__path_*` item must be
   reachable from `openapi.rs`).
2. Add the `#[utoipa::path(...)]` attribute above the handler.
3. Register the handler in `src/openapi.rs` under `paths(...)`.
4. (If it has a named response struct, derive `utoipa::ToSchema` on it and add it
   to `components(schemas(...))`.)

**Shape 1 — no params** (e.g. `workspace`, `roles`, `permissions`):
```rust
#[utoipa::path(get, path = "/v1/orgs/integrations/roles", tag = "integrations",
    security(("workspace_api_key" = [])),
    responses((status = 200, description = "..."), (status = 401), (status = 403)))]
```

**Shape 2 — query params** (e.g. `members`, `activity`, `clients` list): add
`#[derive(utoipa::IntoParams)]` + `#[into_params(parameter_in = Query)]` to the
query struct, then:
```rust
#[utoipa::path(get, path = "...", tag = "integrations",
    params(SomeListQuery), security(("workspace_api_key" = [])),
    responses((status = 200, description = "...")))]
```

**Shape 3 — path param** (e.g. `members/{member_id}`):
```rust
#[utoipa::path(get, path = "/v1/orgs/integrations/members/{member_id}", tag = "integrations",
    params(("member_id" = Uuid, Path, description = "Member UUID")),
    security(("workspace_api_key" = [])),
    responses((status = 200), (status = 404)))]
```

**Body shape — POST/PATCH** (e.g. create client, update role): derive
`utoipa::ToSchema` on the request body struct, add `request_body = TheStruct`:
```rust
#[utoipa::path(patch, path = ".../{member_id}/role", tag = "integrations",
    params(("member_id" = Uuid, Path)), request_body = UpdateMemberRoleRequest, ...)]
```

Response schemas can stay loose (just `status` + `description`) initially and be
tightened to `body = SomeResponse` per endpoint later — the path/params/auth are
what the SDK needs first.

Exit test: `/openapi.json` lists every `/orgs/integrations/*` path with correct
params (respecting `deny_unknown_fields`, `page_size` ≤ 1000); Swagger UI renders
them. Dump the spec without booting the DB via a throwaway example:
`examples/dump_openapi.rs` → `ApiDoc::openapi().to_pretty_json()`.

## Phase C — `@rooiam/sdk-server` (TypeScript, generated + wrapped) ✅ (done)

- [x] Scaffold the monorepo: `rooiam-sdk/packages/js-server` (`@rooiam/sdk-server`)
- [x] Generate base types from `/openapi.json` (`openapi-typescript` →
      `src/generated/schema.ts`; `npm run gen` regenerates)
- [x] Ergonomic wrapper covering the full surface: `workspace()`, `activity()`,
      `auditActions/effectivePolicy/policySummary/roles/permissions/apiKeyMe/widgetPreviewConfig()`,
      `members.{list,get,activity,sessions,revokeSessions,setRole,updateProfile,remove}`,
      `clients.{list,get,create,update,delete,setStatus,secretMetadata,rotateSecret}`,
      `branding.{get,update}`, `authConfig.{get,update}`, `invites.{list,get,send,revoke}`
- [x] API-key Bearer auth, base-URL config, `RooiamError` (status + parsed body),
      query-param helper that omits undefined

**Build note:** generated types must live in `schema.ts` (NOT `schema.d.ts`) —
`tsc` treats `.d.ts` as ambient and won't emit it to `dist`, which would break
`export type { paths } from './generated/schema.js'` for consumers. The `gen`
script outputs `schema.ts`.

Exit test (the stability bar) — **CLEARED**: the suite runs against a **live,
source-run rooiam-server** (port 5180, demo DB) with a real minted workspace API
key. **31/31 tests pass**, including a probe that hits all 11 read endpoints.
Without live env vars the live block skips and 26 unit tests pass. The SDK is
stable + tested against the real server — the bar for refactoring consumers is met.

## Phase E — Refactor consumers onto the proven SDK (one at a time)

Only after C and D pass their stability bars.

- [ ] candycloud-server / candycloud-web (first — already hand-roll it)
- [ ] rooiam-admin (replace hand-rolled `api.ts`)
- [ ] rooiam-app (replace hand-rolled fetch calls)

Each refactor is verified before the next. Payoff: the contract bugs that came
from hand-rolled fetch (oversized `page_size`, unknown-field rejections, wrong
redirect params) become impossible — the SDK types ARE the server's contract.
